| 特性 | 详细说明 |
|------|----------|
| 🛡️ **智能防护引擎** | 云端规则库24小时不间断更新，实时适配各大浏览器、安全软件的拦截策略，自动生成规避规则，确保站点持续可访问 |
| 🕷️ **爬虫白名单机制** | 默认拒绝所有未知爬虫访问，仅放行用户自定义的可信搜索引擎爬虫，避免内容被恶意爬取或违规收录 |
| 🔌 **自动端口映射** | 一键配置端口转发规则，自动封禁内部端口的外部直接访问权限，防止端口穿透攻击，实现内部服务隐藏 |
| 🌐 **Web可视化管理面板** | 直观的后台管理界面，实时展示拦截统计数据，支持一键配置防护规则、端口映射、白名单等所有功能 |

## 📦 快速开始

### 系统要求
- 操作系统：支持 CentOS 7+、Ubuntu 18.04+、Debian 9+ 等主流Linux发行版
- 硬件配置：最低1核1G内存，建议2核2G及以上配置
- 网络环境：服务器需具备ipv4公网IP，且相关端口未被运营商封禁

### 一键安装命令：
在终端执行以下命令即可完成系统安装：
```bash
sudo bash -c "$(curl -fsSL https://raw.githubusercontent.com/AegisProxyX/AegisProxy/main/install.sh)"
```

### 一键卸载命令：
如需完全移除系统，执行以下命令：
```bash
sudo bash -c "$(curl -fsSL https://raw.githubusercontent.com/AegisProxyX/AegisProxy/main/uninstall.sh)"
```

### 安装过程根据提示操作：
1. 输入激活码（联系官方获取）
2. 输入后台端口（直接回车则自动随机生成）
3. 输入后台路径（直接回车则自动随机生成）

### 初始配置

#### 第一步：登录后台
安装完成后，终端会显示后台访问地址，在浏览器中打开该地址，使用激活码登录管理面板。
![AegisProxy展示图](https://raw.githubusercontent.com/AegisProxyX/AegisProxy/main/仪表盘.jpg)

#### 第二步：配置端口映射（必填）
端口映射是实现防护的核心配置，操作步骤如下：
1. 点击左侧菜单栏「端口映射」进入端口管理页面
2. 点击「创建映射」按钮开始添加规则
3. 填写「内部端口」：您的网站或服务实际运行的端口（例如 8000、3000 等）
4. 填写「对外端口」：用户实际访问的端口（例如 80、443 等）
5. 点击「创建映射」完成配置
<img src="https://raw.githubusercontent.com/AegisProxyX/AegisProxy/main/%E7%AB%AF%E5%8F%A3%E6%98%A0%E5%B0%84.jpg" align="left">
<br>
✅ **配置完成后，内部端口会自动禁止外部直接访问，所有流量仅能通过对外端口经由AegisProxy转发。**

## ⚠️ 重要注意事项

### 端口配置规范
 🚨 **内部端口与AegisProxy对外端口必须不同，否则会导致端口冲突，造成服务无法正常访问！**

#### 冲突原因
每个网络端口同一时间只能被一个程序占用，若内部服务端口与AegisProxy对外端口设置为同一端口，两个程序会产生端口抢占，最终导致两者均无法正常运行。

#### 正确配置示例
| 角色 | 端口 | 说明 |
|------|------|------|
| 内部服务（网站/应用） | `8000` | 服务实际监听的本地端口，不对外暴露 |
| AegisProxy（对外端口） | `80` | 用户访问的公开端口，由AegisProxy监听 |

#### 错误配置示例
| 角色 | 端口 | 结果 |
|------|------|------|
| 内部服务（网站/应用） | `80` | ❌ 端口被AegisProxy占用，内部服务启动失败 |
| AegisProxy（对外端口） | `80` | ❌ 端口被内部服务占用，代理服务启动失败 |

#### 核心原则
**内部服务端口与AegisProxy对外端口必须不同，建议内部服务使用1024以上的非常规端口，对外端口使用80/443等标准Web端口。**

## ❓ 常见问题

### Q: 端口映射状态显示"未启动"？
A: 请检查内部端口对应的服务是否正常运行，可在服务器上执行 `curl 127.0.0.1:内部端口` 验证服务可用性。若内部服务未启动，请先启动对应服务后再重启端口映射规则。

### Q: 创建映射时提示"对外端口被占用"？
A: 可使用后台的「端口解除」功能释放被占用端口，或更换其他未被使用的对外端口。也可在服务器执行 `lsof -i:端口号` 查看占用端口的进程，手动结束无关进程。

### Q: 忘记后台登录密码？
A: 后台登录密码即为您的激活码，也可以在SSH命令行查看 输入：AegisProxy

### Q: 程序意外关闭会自动重启吗？
A: 系统默认配置了服务守护进程，当程序意外退出时会在5秒内自动重启，确保持续提供防护能力。

### Q: 系统会限制或检测网站内容吗？
A: **绝对不会。** AegisProxy仅基于浏览器封禁规则进行流量处理，不识别、不检测、不拦截任何站点内容。无论您运行的是企业官网、个人博客、资源站点、游戏服务还是其他类型的应用，系统都会一视同仁，核心目标仅为帮助站点规避浏览器安全拦截，保障正常用户访问。

## ⚡ 规则自动更新机制
系统采用云端-端侧协同的规则更新机制：
1. 各大浏览器、安全软件更新拦截规则后，云端分析系统会自动更新规则
2. 自动生成适配的规避规则并推送到所有已部署的AegisProxy节点
3. 客户端无需手动操作，后台自动完成规则热更新，不影响现有服务运行

即使在您休息时，系统也会持续更新防护规则，确保持续的防护有效性。
